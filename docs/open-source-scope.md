# 开源范围说明

## 文档目的

本仓库并非“完整可复刻式开源”，而是一个面向展示、交流与启发的公开项目。

我希望通过这个仓库完成三件事：
1. 展示项目能力与设计思路
2. 分享部分公开方法论
3. 保留核心资产，避免低成本完全复刻

---

## 本仓库公开的内容

本仓库公开以下内容：

- 项目背景与设计目标
- README 展示页
- 公开版用户使用说明
- 公开版 Prompt 框架
- 效果图、演示图、获奖素材
- 智能座舱与行业应用思考
- 示例输入输出与路线图

这些内容足以帮助外部读者理解：
- 这个项目解决什么问题
- 它与传统天气工具有什么不同
- 它为什么具有 AI 原生体验特征
- 它还能往哪些行业场景延展

---

## 本仓库不公开的内容

以下内容不在本仓库公开：

- 完整系统 Prompt
- 完整风格控制词与调优细节
- `.skill` 技能导出文件
- `.coze` 平台配置文件
- 可直接复刻核心能力的私有工作流
- 失败兜底逻辑与异常处理细节
- 其他内部迭代素材与调试记录

---

## 为什么不全部公开

这个项目的价值并不只在于一个“能跑通的技能”。

真正有积累的部分包括：
- Prompt 工程调优
- 风格稳定性控制
- 城市特色抽取思路
- 天气信息与视觉元素融合方式
- 静态图与动态效果之间的衔接逻辑

如果这些内容被完整公开，这个仓库会更接近“直接复刻资源包”，而不是“展示型开源项目”。

---

## 这意味着什么

你可以通过本仓库：
- 理解项目定位与能力边界
- 看到真实效果与核心思路
- 获得启发并进行二次思考
- 就合作、招聘、方案共创与我联系

但你不能直接通过本仓库获得：
- 完整生产级 Prompt
- 完整 Coze 配置
- 私有工作流导出内容

---

## 关于二次启发与引用

我接受以下方式：
- 基于公开内容获得启发
- 在注明来源的前提下进行讨论与引用
- 基于公开思路做进一步探索

我欢迎交流和二次思考，但不建议将本仓库理解为“完整源码包”或“完整复刻方案”。
