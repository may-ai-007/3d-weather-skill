# 公开版 Prompt 框架

## 文档说明

本文档用于公开说明 3D Weather Skill / SkyVibe Weather 的生成方法框架。

它展示的是“方法结构”，而不是完整可执行 Prompt。
因此，本文档不包含完整系统提示词、完整风格词和内部调优细节。

---

## 总体目标

输入一个城市名，生成具备以下特征的天气视觉内容：

- 反映城市当前天气状态
- 体现城市地标、景观或人文特征
- 呈现统一的 3D 微缩视觉风格
- 输出中文化的信息展示
- 默认可继续生成动态效果

---

## 公开版生成流程

### 1. 用户输入识别
从自然语言中识别用户意图，重点判断：
- 用户是否输入了城市名
- 用户是只想要海报，还是默认海报 + 动态效果
- 用户是否在询问能力介绍，而不是直接请求生成

### 2. 城市与天气信息组织
围绕输入城市组织生成所需的基础信息：
- 城市名称
- 天气状态
- 温度
- 时间或氛围相关信息

### 3. 城市特色抽取
根据城市特点，提炼适合作为视觉主体的内容，包括但不限于：
- 地标建筑
- 自然景观
- 地域文化元素
- 更具识别度的人文特征

### 4. 视觉表达构建
将“天气 + 城市特色 + 中文信息”组合成统一画面结构：
- 主体为城市代表性视觉元素
- 上层叠加天气要素
- 底部保留中文信息区
- 整体留白、质感与光影统一

### 5. 动态效果延展
在默认模式下，基于海报继续生成轻量动态效果。
动态部分强调：
- 与海报内容一致
- 时间短
- 动效自然
- 不依赖复杂分镜与音频

---

## 公开版规则摘要

### 输出模式判断
- 明确说“只要图片 / 海报 / 不要视频”时，仅输出海报
- 其他情况，默认输出海报 + 动态效果

### 文案呈现规则
- 图片上的城市名、天气、温度统一使用中文
- 不显示拼音
- 不显示英文天气词
- 图片中仅保留必要的城市与天气信息，不直接显示用户完整输入

### 风格表达原则
- 视觉风格追求统一、精致和可分享
- 不以传统天气图标为核心
- 更强调 3D 微缩、在地特色、气氛感和视觉完成度

---

## 为什么不公开完整 Prompt

完整 Prompt 不只是描述画面的文本。
它还包含：
- 变量替换规则
- 风格稳定性控制
- 输出一致性约束
- 城市识别与中文化规则
- 失败兜底与异常处理方式

这些部分构成了项目的核心积累，因此不在公开仓库中展示。
